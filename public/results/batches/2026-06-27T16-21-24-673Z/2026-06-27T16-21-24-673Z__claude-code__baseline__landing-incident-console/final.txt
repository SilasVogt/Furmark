## What changed

I replaced the placeholder scaffold with a complete, polished one-page landing site for **Traceboard**, an incident timeline debugger. The page is a static Vite + React + TypeScript app using plain CSS (no UI framework) and `lucide-react` icons.

**New/changed files:**
- `src/App.tsx` — page composition: skip link, nav, hero, stats strip, features, trace deep-dive, workflow, final CTA, footer
- `src/components/IncidentConsole.tsx` — the hero centerpiece: a real app-like "incident console" (correlated swimlane timeline with deploy/alert markers, a root-cause playhead callout, live error-rate + p99 area charts, and a ranked root-cause rail)
- `src/components/TraceWaterfall.tsx` — a distributed-trace span waterfall with a highlighted N+1 bottleneck
- `src/components/Spark.tsx` — lightweight responsive SVG sparklines (the "generated asset," not placeholder boxes)
- `src/components/Nav.tsx` — sticky nav with an accessible disclosure menu (`aria-expanded`/`aria-controls`, Escape-to-close)
- `src/components/Logo.tsx` — a custom incident-pulse SVG brand mark
- `src/data.ts` — content (features, workflow steps, stats, integrations, footer)
- `src/style.css` — full dark "observability tool" design system: tokens, `clamp()`-based responsive type, semantic severity colors, focus-visible rings, reduced-motion support, strict overflow guards
- `src/main.tsx`, `index.html` — entry wiring, plus real `<title>`/meta description/theme-color

**Key fix during development:** the responsive grid overrides used bare `1fr` tracks, so the console's `min-content` was forcing content past the viewport (clipped, not fitting) at 390px. Switching to `minmax(0, 1fr)` and rebuilding the timeline axis as a track-aligned grid resolved both the overflow and the bunched/overlapping timeline labels.
