## Summary

I built a canvas-based **Metrics Data Rain** animation in the existing React + TypeScript + Vite app.

### What changed
- **`src/simulation.ts`** (new) — pure, deterministic physics core. Each metric event becomes a falling bar that accelerates under gravity, collides with the pile in its lane, and stacks on top. When a lane overflows, the oldest (bottom) bar drains and the column drops down (FIFO). No timers/randomness, so it's fully unit-testable.
- **`src/metrics.ts`** (new) — six metrics (CPU/MEM/NET/REQ/LAT/ERR), each given a unique **color + hatch pattern + short text code** so meaning never depends on color alone.
- **`src/patterns.ts`** (new) — builds the per-metric texture tiles (diagonal, dots, grid, vertical, etc.).
- **`src/App.tsx`** (new) — canvas renderer + controls: a simulated feed (random-walk values), **Pause/Resume** (Space), **Reduced motion** toggle (R, auto-on if the OS prefers it), feed-rate select, Burst, Clear, a live feed log, and an on-canvas HUD. Includes DPR scaling and a `ResizeObserver`.
- **`src/main.tsx`**, **`src/style.css`** — wired up the entry point and added the dashboard styling.
- **`src/simulation.test.ts`** (new) — 10 tests covering each success criterion.
