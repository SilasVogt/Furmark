## What changed

The fixture's batching algorithm was already correct (layered Kahn's algorithm with alphabetical sorting and referenced-only deps), but it had one real spec gap: on a cycle it reported **every** unscheduled node as a cycle member — including nodes that merely *depend on* a cycle. For `{a:["b"], b:["a"], c:["a"]}` it named `["a","b","c"]` when only `a` and `b` form the cycle.

**`src/dependencyBatches.ts`**
- Replaced the cycle diagnostic with a `findCycleMembers` helper using **Tarjan's strongly connected components**, so `DependencyCycleError.cycle` now names *only* genuine cycle members (SCCs of size ≥ 2, plus self-loops), sorted alphabetically.
- Kept the layered Kahn batching, switching the per-node dependency `Set` to an immutable edge map (for cycle reporting) plus a mutable `remaining` count, so self-loops (`a:["a"]`) are correctly detected rather than masked.

**`tests/dependencyBatches.test.ts`**
- Kept the original four cases and expanded to 13, covering: empty graph, diamond, multi-level depth ordering, insertion-order determinism, referenced-only deps (single + shared), disconnected graphs/independent chains, and cycles — two-node, longer (`a→b→c→a`), self-loop, a node depending on but not in the cycle, and multiple disjoint cycles. The cycle tests also assert the structured error's `instanceof`, `.cycle`, and `.message`.
